## ✨ Features

- ✅ Organization-wide Issue Templates  
- ✅ Structured Issue Forms (YAML-based)  
- ✅ Standardized Pull Request Template  
- ✅ Automatic availability across all repos  
- ✅ Improved developer experience and contribution quality  

---

## 🧩 Issue Templates

Located in: `.github/ISSUE_TEMPLATE/`

### 🐞 Bug Report (`bug_report.yml`)
Used to report bugs with structured fields such as:
- Description  
- Steps to reproduce  
- Expected vs actual behavior  

---

### 🚀 Feature Request (`feature_request.yml`)
Used to suggest improvements or new features:
- Problem statement  
- Proposed solution  
- Additional context  

---

### ⚙️ Configuration (`config.yml`)
Controls issue behavior:
- Disable blank issues  
- Provide support/contact links  
- Guide users to correct channels  

---

## 🔁 Pull Request Template

**File:** `.github/PULL_REQUEST_TEMPLATE.md`

Ensures every PR includes:
- Summary of changes  
- Type of change (bug, feature, etc.)  
- Checklist for contributors  
- Related issue references  

---

## 🌍 How It Works

- This repository is named **`.github`**, which GitHub treats as a **special global configuration repo**
- Templates defined here are automatically applied to:
  - All repositories in the organization  
- Individual repositories can override these templates if needed  

---

## 🚀 Usage

No additional setup is required.

Once this repository exists in your organization:

1. Go to any repository  
2. Create a new Issue or Pull Request  
3. The templates will automatically appear  

---

## 🛠 Customization

To modify templates:

1. Update the relevant `.yml` or `.md` file  
2. Commit changes to the default branch  
3. Changes are instantly reflected across all repositories  

---

## ⚠️ Best Practices

- Keep templates **simple and focused**  
- Use **required fields** to enforce quality  
- Add **labels** for better issue tracking  
- Regularly update templates based on team feedback
